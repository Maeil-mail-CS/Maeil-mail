🔗 질문 링크: [동일성과 동등성에 대해서 설명해주세요.](https://www.maeil-mail.kr/question/71)

✅ 답변 내용:
<pre>
  자바를 기준으로 설명하자면 동일성은 두 객체가 완전히 같은 것(즉, 참조값이 동일) 
  동등성은 두 객체가 각각 독립적으로 존재하지만 객체가 다루는 정보만 같고, 참조값은 다른 것
  (Ex. User 클래스의 객체 user1{name: 한얼}, user2{name: 의연}처럼 동등성을 가짐)
</pre>

💡 꼬리 질문1: `equls()`와 `==`의 차이는 무엇인가요?
<pre>
  equls()는 객체의 내용을 비교, == 연산자는 참조값을 비교
</pre>


💡 꼬리 질문2:  String은 객체인데 `==` 비교해도 되던데 어떻게 된건가요?
<pre>
  String 객체를 만드는 방법이 2가지
  new 연산자로 생성, 리터럴로 생성하는 방식이 있다.
  (Ex. String str = new String("의연최고"), String str = "의연 최고")

  new로 만들면 같은 내용을 가진 문자열이어도 아예 새로운 heap공간에 객체가 생성
  리터럴로 만들면 String Pool에 저장이 되기 때문에 같은 내용을 가진 문자열이라면 동일한 객체를 참조한다.

</pre>

💡 꼬리 질문3:  Integer 같은 래퍼 클래스는 어떻게 비교하나요?
<pre>
  래퍼 클래스는 객체이므로 내용을 비교하고 싶은 경우  equls(), 참조를 비교하고 싶은 경우 == 
</pre>

📝 피드백 내용:
<pre>
  꼬리 질문2(String은 객체인데 `==` 비교해도 되던데 어떻게 된건가요?) 에서 
  구체적인 예시를 들어 new 연산자로 생성하는 것과 문자열 리터럴로 생성하는 방식을 설명했는데
  예시를 들어서 설명하다보면 답변이 장황해지므로 간략하게 답변하기
</pre>
